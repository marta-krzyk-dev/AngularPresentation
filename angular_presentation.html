Angular is an open-source web application framework developed by Google and world-wide community. The project's base was previous versions of Angular that is reffered to as AngularJS. In this presentation will touch upon Angular2+. AngularJS and Angular2+ are driven by different solution styles.
Use space or arrows on keyboard to navigate :)
Technologies
Angular is a platform and framework for building single-page client applications using HTML and TypeScript. CSS is used to apply style onto HTML.
Angular is written in TypeScript which is a superset of JavaScript that adds static typing, interfaces, generics, enumerated types and more.
Modulesand components
Angular app is a set of NgModules: at least 1 root module (AppModule) and any number of feature modules. The children of a module are called components. NgModules (parents) provide a compilation context for their components.
Take a look at the root folder of an Angular app. The directories help to mark the hierarchy between the modules and components.
component
is made up of 3 things: a TypeScript class, HTML view and CSS style.
Look at the Product component class below. You define a component's application logic—what it does to support the view—inside a class. The class interacts with the view through an API of properties and methods. For example, ProductListComponent has a products property that holds an array of products.
The @Component decorator identifies the class immediately below it as a component class, and specifies its metadata.
Selector : A CSS selector that tells Angular to create and insert an instance of this component wherever it finds the corresponding tag in template HTML.
Template Url : The module-relative address of this component's HTML template.
view
A component controls a patch of screen called a view. You define a component's view with its companion template. A template is a form of HTML that tells Angular how to render the component.
Template looks like regular HTML but it contains Angular template syntax, which alters the HTML. Your template can use data binding, pipes to transform data before it is displayed, and directives to apply app logic to what gets displayed.

Directives
Directives are instructions used by Angular to transform DOM. A directive is a class with a @Directive() decorator.
Structural directive
Structural directives alter layout by adding, removing, and replacing elements in the DOM. The most popular ones are: *ngFor, *ngIf, ngSwitch (works together with *ngSwitchCase, *ngSwitchDefault).
Attribute directive
Attribute directives alter the appearance or behavior of an existing element.
You can also write your own directives!
service
  • Service is any value, function, or feature that an app needs.
  • A component can delegate certain tasks to services, such as fetching data from the server, validating input, or logging
  • Services are injected into components via dependency injection.
Pipe
Pipes are simple functions you can use in templates to transform and format strings, currency amounts, dates, and other display data.
You can chain pipes like so: {{ birthday | date | uppercase}} !

Let's summarize...
Cons
  • Constant framework updates add tons of work to the developers' plate
  • Steep learning curve
  • Big and compilated
  • The lifecycle app is complex and time-consuming to master
  • Poor scalability
  • Bad performance when rendering big data grids, lists which leds to UI lags*
  • Name collisions are common
*The culprit is so-called dirty checking, which is scanning scope for changes in variables. This can be time-ineffective.
Pros
  • Angular directly modifies DOM (it's fast!)
  • Bindings between DOM elements and TypeScript code
  • Hiding implementation details under the hood
  • Reusing reliable, ready-to-go components
  • Graphical templates of are more readable than when using other technologies, eg. jQuery
  • Decoupling application’s logic from DOM manipulation
